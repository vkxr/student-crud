PART 1: PROJECT OVERVIEW

What does this app do?

This is a Student Manager — a CRUD application:

  • Create — add new students via a modal form
  • Read — view all students in a table
  • Update — edit existing students (form pre-fills)
  • Delete — remove students with confirmation dialog

Plus: search/filter, Excel export, and localStorage persistence.

Interview Answer: "Walk me through this project"

"I built a Student Manager using React with TypeScript and Tailwind CSS. It lets you add, edit, and delete students from a table. The data persists using localStorage so it survives page refreshes. I also added search filtering and Excel export using the xlsx library. On the backend side, I built a REST API with Express, TypeScript, Prisma ORM connected to a Neon PostgreSQL cloud database, with Zod schema validation for incoming requests."

📄 types/student.ts — TypeScript Interfaces
export interface Student {
  id: number
  name: string
  email: string
  age: number
  createdAt?: string
}

export interface StudentFormData {
  name: string
  email: string
  age: number
}

What is an interface?

An interface defines the shape of an object. It tells TypeScript: "any object of this type MUST have these fields with these types."

Why two interfaces?

  • Student has id and optional createdAt — a saved student
  • StudentFormData only has name, email, age — what the form submits (ID is generated separately)

Interview answer:

"I created two interfaces because the form doesn't know the student's ID when submitting — the ID is generated by Date.now() on the frontend or autoincrement() on the database. Separating them makes the types more precise."

What does ? mean?

createdAt?: string means this field is optional — it might exist or might not.

localStorage Helper Functions

const STORAGE_KEY = 'students_data'

function loadStudents(): Student[] {
  const saved = localStorage.getItem(STORAGE_KEY)
  if (saved) {
    return JSON.parse(saved)
  }
  return initialData as Student[]
}

function saveStudents(students: Student[]) {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(students))
}

What is localStorage?

  • localStorage is a browser API that stores key-value pairs
  • Data persists even when you close the browser
  • It only stores strings, so we JSON.stringify() to save and JSON.parse() to load
  • Maximum ~5MB per domain

How does the fallback work?

  1. Try to read from localStorage
  2. If nothing stored (first visit), use JSON file data
  3. Every time students change, save to localStorage
"localStorage gives us persistence without a backend. I save the students array as a JSON string whenever it changes, and load it back on page load. If it's the first visit, I fall back to the initial JSON data."

State Variables

const [students, setStudents] = useState<Student[]>([])
const [loading, setLoading] = useState(true)
const [searchQuery, setSearchQuery] = useState('')
const [isFormOpen, setIsFormOpen] = useState(false)
const [editingStudent, setEditingStudent] = useState<Student | null>(null)
const [deleteTarget, setDeleteTarget] = useState<Student | null>(null)

What is useState?

useState is a React Hook that lets you add state to functional components. Returns a value and a setter function.

const [value, setValue] = useState(initialValue)

What is <Student[]>?

A generic type parameter. Tells TypeScript the state will hold an array of Student objects.

What is Student | null?

A union type. The value can be either a Student object OR null. Used for editingStudent because:

  • No student being edited → null
  • Editing → the actual Student object

Why not use Redux?

"For a small app like this, useState is enough. Redux adds complexity (actions, reducers, store) that isn't needed when all state lives in one component. I'd use Redux or Zustand if the app grew larger."

useEffect #1 — Loading Data on Mount

useEffect(() => {
  const timer = setTimeout(() => {
    setStudents(loadStudents())
    setLoading(false)
  }, 1500)
  return () => clearTimeout(timer)
}, [])

What is useEffect?

useEffect runs side effects — things outside React's rendering cycle like fetching data, timers, subscriptions.

What does [] (empty array) mean?

The dependency array. [] means "run only once, when component mounts." [searchQuery] would run every time searchQuery changes.

What is the cleanup function?

return () => clearTimeout(timer) runs when the component unmounts. Prevents memory leaks if user navigates away before the 1.5s timer finishes.

"I use useEffect with an empty dependency array to load data once on mount. The 1.5-second delay simulates an API call. The cleanup function cancels the timer if the component unmounts early, preventing memory leaks."

useEffect #2 — Persisting to localStorage

useEffect(() => {
  if (!loading) {
    saveStudents(students)
  }
}, [students, loading])

Why the if (!loading) check?

Without it, the first render would save an empty array [] to localStorage before data loads.

Why [students, loading]?

Re-runs whenever students or loading changes. Every add/edit/delete changes students → effect runs → saved to localStorage.

Filtering Logic

const filteredStudents = students.filter((s) => {
  const q = searchQuery.toLowerCase()
  return s.name.toLowerCase().includes(q) || s.email.toLowerCase().includes(q)
})

What does .filter() do?

Creates a new array containing only elements that pass the test. Doesn't modify the original.

Why .toLowerCase()?

Makes search case-insensitive. Without it, "rahul" wouldn't match "Rahul".

"I derive filteredStudents from the students array. It's not stored in state because it can be computed. This follows React's 'derived state' principle — don't store what you can calculate."

CRUD Handler — Add

const handleAdd = (data: StudentFormData) => {
  const newStudent: Student = {
    ...data,
    id: Date.now(),
    createdAt: new Date().toISOString(),
  }
  setStudents((prev) => [...prev, newStudent])
  setIsFormOpen(false)
}

What is the spread operator ...?

...data copies all properties from data into the new object.

Why Date.now() for ID?

Returns current timestamp in milliseconds. Quick unique ID without a library. In production, the database generates IDs.

What is the functional update pattern?

setStudents((prev) => [...prev, newStudent]) — passes a function that receives previous state. Safer when multiple updates happen simultaneously.

CRUD Handler — Update

const handleUpdate = (data: StudentFormData) => {
  if (!editingStudent) return
  setStudents((prev) =>
    prev.map((s) => (s.id === editingStudent.id ? { ...s, ...data } : s))
  )
  setEditingStudent(null)
  setIsFormOpen(false)
}

What does .map() do here?

Creates new array. For each student: if ID matches → replace with updated data; otherwise → keep original.

Why { ...s, ...data }?

data has name, email, age. Student also needs id, createdAt. Spread copies original fields first, then overwrites with form data.

"I use .map() to create a new array where only the matching student is updated. This is an immutable update pattern — I never mutate the original array directly. React needs new references to detect changes."

CRUD Handler — Delete

const handleDelete = () => {
  if (!deleteTarget) return
  setStudents((prev) => prev.filter((s) => s.id !== deleteTarget.id))
  setDeleteTarget(null)
}

.filter() returns a new array with all students except the one with the matching ID.

const validate = (): Record<string, string> => {
  const errs: Record<string, string> = {}
  if (!name.trim()) errs.name = 'Name is required'
  if (!/\S+@\S+\.\S+/.test(email)) errs.email = 'Enter valid email'

What is Record<string, string>?

What does the email regex do?

/\S+@\S+\.\S+/ checks: something@something.something. Not perfect but reasonable for assignments.

What is e.preventDefault()?

Stops the browser's default form submission (page reload) so React handles it in JavaScript.

"The form works in two modes — add and edit. useEffect watches editingStudent and pre-fills fields when editing. Validation runs on submit, returning an errors object displayed as inline messages."

What is a controlled component?

Controlled: React state drives the input's value. Changes go through onChange → parent updates state → re-render.

Uncontrolled: Input manages its own value (using ref). We use controlled for predictability.

📄 utils/exportExcel.ts — Excel Export
export function exportToExcel(students: Student[], filename = 'students'): void {
  // 1. Map to clean objects (keys become column headers)
  const data = students.map((s) => ({ Name: s.name, Email: s.email, Age: s.age }))

  // 2. Create worksheet and workbook
  const worksheet = XLSX.utils.json_to_sheet(data)
  const workbook = XLSX.utils.book_new()
  XLSX.utils.book_append_sheet(workbook, worksheet, 'Students')

  // 3. Write as ArrayBuffer with correct MIME type
  const excelBuffer = XLSX.write(workbook, { bookType: 'xlsx', type: 'array' })
  const blob = new Blob([excelBuffer], {
    type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
  })

  // 4. Trigger download via hidden anchor
  const url = URL.createObjectURL(blob)
  const link = document.createElement('a')
  link.href = url
  link.download = `${filename}.xlsx`
  document.body.appendChild(link)
  link.click()
  document.body.removeChild(link)
  URL.revokeObjectURL(url)
}

Step by step:

  1. Map data → capitalized keys become column headers in Excel
  2. json_to_sheet → converts array to Excel worksheet
  3. book_new + book_append_sheet → creates workbook with sheet
  4. XLSX.write → outputs as ArrayBuffer (raw binary)
  5. new Blob → wraps with correct MIME type so OS knows it's an Excel file
  6. URL.createObjectURL → temporary URL pointing to the blob
  7. Hidden anchor trick → creates <a>, sets download attribute, clicks programmatically
  8. Cleanup → removes link and revokes URL

What is a Blob?

Blob (Binary Large Object) = raw data. The MIME type tells browser/Excel what the file format is.

"I use xlsx to convert arrays into Excel worksheets. I create a Blob with the correct MIME type and trigger a programmatic download instead of using writeFile(), which can have MIME issues."

What is Express?

Minimal web framework for Node.js that handles HTTP requests and responses.

What is middleware?

What is CORS?

Cross-Origin Resource Sharing. Browsers block requests between different origins by default. CORS sends headers saying "allow this."

📄 src/schemas/studentSchema.ts — Zod Validation
import { z } from 'zod'

export const createStudentSchema = z.object({
  name: z.string().min(1, 'Name is required').max(100),
  email: z.string().min(1, 'Email is required').email('Valid email required'),
  age: z.number().int().min(1).max(120),
})

export const updateStudentSchema = createStudentSchema.partial()

export type CreateStudentInput = z.infer<typeof createStudentSchema>
export type UpdateStudentInput = z.infer<typeof updateStudentSchema>

What is Zod?

TypeScript-first schema validation library. Defines rules and validates at runtime.

Why not just TypeScript types?

TS types only exist at compile time — erased when code runs. Someone could send { name: 123 } and TS wouldn't stop it. Zod validates at runtime.

What is z.infer?

Automatically creates a TypeScript type from the Zod schema. No need to define the type separately.

What is .partial()?

Makes all fields optional. Used for updates where you might only want to change one field.

What is safeParse()?

const parsed = createStudentSchema.safeParse(req.body)
if (!parsed.success) {
  res.status(400).json({ error: parsed.error.errors[0].message })
  return
}

Returns { success: true, data } or { success: false, error } without throwing. Safer than .parse().

"I use Zod for runtime validation because TypeScript types are erased at runtime. Zod defines validation rules like min length, email format, numeric ranges. safeParse validates without throwing exceptions."
📄 src/controllers/studentController.ts — CRUD Handlers
export const createStudent = async (req: Request, res: Response) => {
  try {
    // 1. Validate with Zod
    const parsed = createStudentSchema.safeParse(req.body)
    if (!parsed.success) {
      res.status(400).json({ error: parsed.error.errors[0].message })
      return
    }

    // 2. Database operation with Prisma
    const { name, email, age } = parsed.data
    const student = await prisma.student.create({
      data: { name, email, age },
    })

    // 3. Send response
    res.status(201).json(student)
  } catch (error: any) {
    // 4. Handle specific Prisma errors
    if (error.code === 'P2002') {
      res.status(400).json({ error: 'Email already exists' })
      return
    }
    res.status(500).json({ error: 'Failed to create student' })
  }
}

PART 5: COMMON INTERVIEW QUESTIONS

Q: "Why React instead of Angular/Vue?"

"React is the most adopted UI library with the largest ecosystem. Component-based, virtual DOM for performance, great TypeScript support."

Q: "Why Vite instead of Create React App?"

"CRA is no longer maintained. Vite uses native ES modules for instant hot reload and Rollup for optimized builds."

Q: "Why localStorage instead of a database?"

"For a frontend assignment, localStorage provides persistence without backend setup. The architecture lets us switch to API calls by just changing the data source."

Q: "Why Tailwind instead of regular CSS?"

"Tailwind lets me style in JSX without switching files. Utility-first approach: compose classes like bg-blue-600 text-white px-4."

Q: "How does search filtering work?"

"It's a derived value — computed from students and searchQuery on every render. Case-insensitive matching on name and email."

Q: "How would you scale this?"

"Add React Query for data fetching with caching, pagination, authentication, Zustand for global state if multi-page."

Q: "Why Zod instead of manual validation?"

"Zod gives declarative validation with auto-generated types via z.infer. TypeScript types are compile-time only — Zod validates at runtime."

Q: "How does the Excel export work?"

"I use xlsx to create worksheets from JSON, write to ArrayBuffer, wrap in a Blob with correct MIME type, and trigger download via a hidden anchor element."

Q: "Why Prisma instead of raw SQL?"

"Prisma provides type-safe queries with autocomplete, automatic migrations, and catches schema mismatches at compile time."

Q: "What is the difference between PUT and PATCH?"

"PUT replaces the entire resource. PATCH updates specific fields. I used PUT since we send all fields in the update form."
